Much of my research is about applying machine learning in embedded systems to bridge the gap, addressing the challenges in downstream tasks, security, bug detection, and explanation. Representative papers are highlighted.

Are Emily and Greg Still More Employable than Lakisha and Jamal? Investigating Algorithmic Hiring Bias in the Era of ChatGPT
Akshaj Kumar Veldanda, Fabian Grob, Shailja Thakur, Hammond Pearce, Benjamin Tan, Ramesh Karri, Siddharth Garg
NeurIPS   (FoMo Workshop), 2023

This project is evaluating large language models like GPT-3.5, Bard, and Claude for bias in resume screening tasks. Specifically, it is testing these models on matching resumes to job categories and summarizing resumes, while looking for biases related to gender, race, pregnancy status, and political affiliation. The goal is to measure the extent of algorithmic bias in AI systems being considered for automated hiring workflows.

From Concepts to Reality: Bridging Gaps in ML-Guided EDA with Curated Datasets
Animesh Chowdhury, Shailja Thakur, Hammond Pearce, Ramesh Karri, Siddharth Garg
ICCAD   (Special Session Proposal), 2023

This paper highlights the lack of standard datasets for ML-guided EDA tools in the journey from RTL to GDSII and introduces two high-quality datasets: Verigen, derived from GitHub and Verilog textbooks for Verilog code generation, and OpenABC-D, for logic synthesis tasks. The discussion encompasses challenges in dataset curation and growth, quality, security, and the application of specialized data augmentation tools for the hardware domain.

VeriGen: A Large Language Model for Verilog Code Generation
Shailja Thakur, Baleegh Ahmad, Hammond Pearce, Benjamin Tan, Ramesh Karri, Brendan Dolan-Gavitt, Siddharth Garg
  (under review), 2023

This is the first study to comprehensively explore both commercial and open-source Large Language Models (LLMs) for their Verilog code generation capabilities. By comparing these with our in-house trained LLM for Verilog, the results demonstrate that our model exhibits superior performance, underlining the potential of specialized LLMs in hardware design automation.

LLM-assisted Generation of Hardware Assertions
Rahul Kande, Hammond Pearce, Benjamin Tan, Brendan Dolan-Gavitt, Shailja Thakur, Ramesh Karri, Jeyavijayan Rajendran
  (under review), 2023

In this work, we investigate the use of emerging large language models (LLMs) for code generation in hardware assertion generation for security, where primarily natural language prompts, such as those one would see as code comments in assertion files, are used to produce SystemVerilog assertions.

On Hardware Security Bug Code Fixes By Querying Large Language Models
Baleegh Ahmad, Shailja Thakur, Benjamin Tan, Ramesh Karri, Hammond Pearce
  (under review), 2023

In this research, we harness the power of Large Language Models (LLMs) to automatically detect and rectify security-related bugs in Verilog hardware designs. By curating a unique dataset and employing a tailored framework, our method surpasses existing automated tools in hardware bug repair.
